## 课题二：多模态数据对齐驱动的跨模态信息融合技术研究

### 1.1 基于动态稀疏路由的多模态增量对齐技术

本项目拟提出一种基于动态稀疏路由的多模态增量对齐技术，旨在实现对海量多模态数据的高效语义对齐。本项目拟从以下几个方面展开研究：首先，构建**动态稀疏路由**机制，借鉴条件门控思想，设计模态感知的路由网络，仅激活与当前输入相关的局部对齐模块（如图像区域-文本实体对、音频片段-短语对），通过参数隔离策略有效降低新增模态适配器的训练成本。其次，引入**增量式对齐学习**框架，融合ChatVLA的阶段化训练策略，在冻结共享主干网络的基础上，采用“先全局锚定、再局部微调”的渐进式对齐方式，实现新模态的零样本快速扩展，避免全量模型的重复训练。基于以上核心技术，本项目能够突破传统多模态对齐的“全量训练、静态对齐”的局限性，实现细粒度场景下的高效训练。

### 1.2 面向长序列的动态嵌入与层次对齐优化技术​

本项目拟提出一种面向时间序列模态（如视频与音频）的高效对齐技术。首先，本项目将实现**分层序列压缩**方法，采用区域级token压缩算法，对长视频或语音序列进行关键帧提取与语义聚类，在保持细粒度对齐能力的同时，压缩token数量，防止上下文窗口的溢出。其次，本项目拟探索构建**局部–全局联合对齐**机制，在局部层面引入基于残差向量量化的音频 Token 与视频块 Token 掩码重建任务，约束跨模态同步精度，在全局层面添加事件语义匹配损失与单调连续性约束，避免特征漂移。最后，本项目拟通过**动态时间嵌入​**​，将视频帧率与语音采样率映射到统一时间轴，实现每帧与音频片段的精确自适应对齐，并引入分段线性时间规整自适应修正时延偏差。基于以上核心技术，本项目将充分考虑时间序列模态的时空特性，实现长序列模态下端到端多模态对齐的高效训练。

### 拟解决的关键问题

**关键问题：模态细粒度对齐的训练效率问题**
在多模态数据预处理系统中，实现非语言模态与文本的细粒度对齐是构建高质量融合表征的重要一环。与仅关注整体语义匹配的全局对齐不同，细粒度对齐要求将图像中的局部区域、语音帧或视频片段与文本中的具体实体或短语建立精确对应关系。此外现有方法通常仅能同时处理两到三种模态。一旦引入更多新模态便需进行大规模的对齐预训练或指令微调，从而带来十分高昂的计算与存储成本，严重制约训练的效率。此外，新增模态时对共享参数的调整又会引发灾难性遗忘，会导致之前已训练好的模态知识被遗忘，进一步降低训练的效率。针对上述关键问题，本项目拟通过基于动态稀疏路由的多模态增量对齐技术，实现跨模态细粒度对齐的高效训练。

**关键问题：时间序列模态对齐的溢出和错配问题**
在长序列视频与语音的对齐训练中，随着帧数和语音片段的激增，多模态输入的 token 数量往往远超模型的上下文窗口容量，导致无法准确完成跨模态同步。此外，不同模态在采样率、时延或动态变化特性上的差异，会引入隐式的时序错位，使得同一事件在视频和音频中的时间戳不再一一对应。现有方法往往难以同时兼顾动态时间依赖建模与实时对齐，容易遗漏跨模态局部动态的因果约束，从而引发表征学习中的特征漂移，进而在多模态融合和下游检索等任务中带来噪声与语义歧义。本项目旨在提出动态嵌入与层次对齐优化技术​，以解决长序列多模态数据在细粒度对齐过程中的上下文溢出与时间错配难题。

**关键问题：跨模态融合的语义鸿沟和模态偏差问题**
不同模态对同一语义的表达存在本质差异，图像以像素和纹理呈现细粒度视觉特征，文本以离散符号编码抽象概念，语音和视频则融汇时序与空间信息。这种“语义鸿沟”使得简单的特征拼接或加权难以实现有效融合。与此同时，多模态数据来源异构、标注质量不一，噪声标注、模态缺失或分布失衡（如文本缺失、图像模糊、时序采样不完整）会削弱融合效果，导致模型过度依赖高频特征，难以跨域泛化。低资源模态普遍依赖合成数据，生成偏移又可能引入虚假关联，进一步加剧理解偏差；不同模态间的数据量与编码器性能差异，又使模型易偏向主导模态，忽视弱模态中的判别信息。针对上述挑战，本项目拟构建xxx，进而提升系统的鲁棒性与跨域推理能力。

**5.技术挑战**
- **模态拓展**： 只能处理2到3种非语言模态，在引入更多模态时面临挑战
	- **训练效率问题**
		- 通过新增对齐预训练和指令微调来引入新模态，但这种方式会带来高昂的训练成本
		- 虽然可以借助Specific-MLLM（特定模态模型）的已有知识，或使用预对齐编码器进行间接对齐来降低训练开销，但这可能会影响跨模态的理解效果
	- **灾难性遗忘**
		- 新增模态调整共享参数时，可能会导致之前已训练好的模态知识被遗忘
		- 可以通过混合已训练模态的数据，或仅微调模态专用参数来部分缓解，但使训练过程更加复杂
	- **低资源模态问题** 
		- 罕见模态（如红外图像、特定传感器数据）缺乏足够真实数据，只能靠合成数据替代，可能导致理解偏差
- **跨模态能力**
	- **长上下文处理** 
			- 多个序列模态（如视频、语音）时，多模态token的数量可能超出窗口限制
		- 通过token压缩或token采样，来减少输入token数量，但也会导致跨模态性能下降
	- **模态偏差**
		- 不同模态训练数据量不均衡以及各模态编码器性能差异，更关注主导模态，而忽略其他模态的信息
		- 可能要平衡各模态的数据量或增强相应模态模块
	- **时间对齐问题**
		- 处理具有时间依赖性的多模态信息时的时间对齐信息
		- 交错插入音视频模态token，或在多模态token中插入时间相关的特殊标记

		- 传统逐条比对的方式效率极低。通过统一编码和优化搜索，系统能在毫秒级响应复杂查询，同时保证结果精准
	- **多模态编码器**： 融合、对齐等知识
- **模态检索模式**
	- **文本模态**： 从关键词到语义理解
		- _BM25_：（类似搜索引擎的关键词匹配）快速筛选相关文本
		- _MiniLM/BGE-M3_：用深度学习模型捕捉语义，找到"意思相近但措辞不同"的内容
		- _ColBERT/PreFLMR_：分析词语间关系，提升复杂问题（如医疗术语）的精准度
		- _RAFT/CRAG_：精准标注文本来源段落，方便追溯答案依据
	- **图像检索：** 从像素匹配到跨模态搜索
		- _CMI模型_：融合颜色、形状等多特征，支持"找蓝色圆形logo"等组合条件（图片特征）
		- _Pic2word_：把图片自动转文字描述，实现用文字搜图（内容）
		- _EchoSight_：结合百科知识，理解图片背后的概念（标签分类）
	- **视频检索：** 时间维度+多模态融合
		- _iRAG_：分段分析长视频，逐步理解剧情发展
		- _RTime_：用反向视频片段排除干扰，专注关键动作（如体育赛事精彩瞬间）
		- _Video-RAG_：结合视频中的文字（字幕/OCR文字）和画面，支持"找主角说某台词的片段"
		- _T-Mass_：把文本描述转化为动态特征，提升"校园生活vlog"等模糊搜索的准确性
- **重排名策略**
	- **优化示例选择**：通过多步检索整合监督与非监督策略，提升上下文质量
	- **相关性评分优化​**：结合多模态相似性评估和深度学习模型，精细化排序
	- **过滤机制​**：通过主动剔除低质数据，增强系统鲁棒性​
